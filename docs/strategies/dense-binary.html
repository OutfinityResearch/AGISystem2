Dense Binary Strategy

← Back to Documentation

The dense binary strategy is the default HDC implementation in AGISystem2, using fixed-dimensional vectors and bitwise operations.

Overview

The dense binary strategy represents vectors as fixed-size bit arrays, typically using 32,768 bits (4KB) per vector. This approach is mature, well-tested, and offers excellent performance for most applications.

Key Characteristics

  • Binding Operation: Bitwise XOR (⊕)
  • Bundle Operation: Majority vote per bit position
  • Similarity: 1 - (Hamming distance / geometry)
  • Memory: 4KB per vector (32,768 bits)

Mathematical Properties

  • Self-Inverse: bind(bind(a,b), b) = a (perfect)
  • Associative: bind(bind(a,b), c) = bind(a, bind(b,c)) (perfect)
  • Commutative: bind(a,b) = bind(b,a) (perfect)
  • Reflexive: similarity(a,a) = 1.0 (perfect)
  • Symmetric: similarity(a,b) = similarity(b,a) (perfect)

Performance

Operation Complexity Performance
Binding O(n/32) ~255K ops/sec
Similarity O(n/32) ~145K ops/sec
Bundle O(n/32) ~948 ops/sec

Use Cases

  • General-purpose applications
  • Smaller knowledge bases (<10K facts)
  • Applications requiring maximum performance
  • Backward compatibility scenarios

Non-monotonic Logic

Back to index | Back to concepts

Non-monotonic logic allows conclusions to be withdrawn when new information arrives; AGISystem2 supports this behaviour through layers and overrides that can retract default inferences without rewriting the entire knowledge base.

In AGISystem2

The system implements non-monotonic reasoning through its theory layer system, which allows for the creation of contexts where default assumptions can be overridden by more specific information.

Key non-monotonic reasoning features:

  • Default reasoning - Making assumptions in the absence of contradictory information
  • Exception handling - Managing cases where general rules don't apply
  • Belief revision - Updating conclusions when new information arrives
  • Layered reasoning - Using context-specific overrides to handle exceptions

Technical Implementation

Non-monotonic logic is implemented through the TheoryLayer module, which maintains a stack of contexts where each layer can override or modify conclusions from lower layers. This allows for efficient belief revision without requiring complete recomputation of the knowledge base.

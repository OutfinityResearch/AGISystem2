Deontic Logic

Back to index | Back to concepts

Deontic logic studies permissions, obligations and prohibitions; in AGISystem2 these ideas live on dedicated value dimensions and are manipulated through deontic relations that mark actions as allowed, required or forbidden in a given context.

In AGISystem2

The system implements deontic reasoning through specialized dimensions and relations in the vector space. This allows for the representation and manipulation of normative concepts like permissions, obligations, and prohibitions.

Key deontic concepts include:

  • Deontic dimensions - Specific axes that encode normative information
  • Obligation bands - Regions representing required actions or states
  • Permission bands - Regions representing allowed actions or states
  • Prohibition bands - Regions representing forbidden actions or states

Technical Implementation

Deontic reasoning is implemented through the Reasoner module, which can evaluate statements against deontic criteria and generate explanations for normative decisions. The system uses geometric operations to determine whether actions fall within permitted, obligated, or forbidden regions of the conceptual space.

Efficient ML Execution Trends

A survey of optimized runtimes, compilers, and libraries.

Optimized Inference Runtimes

ONNX Runtime Microsoft's cross-platform engine using Execution Providers (EP) like OpenVINO or XNNPACK.
OpenVINO Intel's toolkit for maximizing performance on Intel silicon using INT8 quantization and IR conversion.
TensorFlow Lite Google's framework for mobile/edge, using FlatBuffers and XNNPACK for low-footprint inference.
Core ML Apple's on-device framework that seamlessly falls back to CPU when the Neural Engine isn't available.

ML Compilers

Compilers like Apache TVM and XLA produce optimized machine code, closing the gap between high-level frameworks and raw hardware performance through operator fusion and auto-tuning.

Low-Level Kernel Libraries

  • oneDNN: Intel's foundational library for deep learning building blocks.
  • XNNPACK: Google's highly optimized solution for ARM, x86, and WebAssembly.
  • FBGEMM: Meta's library for high-performance low-precision server-side inference.

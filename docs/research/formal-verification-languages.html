Formal Verification Languages

Proving that systems work as intended, before a single line of code is run.

System Design as Science

Formal verification involves using mathematical methods to prove that a system (hardware, software, or an AI agent) satisfies a set of specifications. This is the foundation of Auditable System 2 Engineering.

Key Modeling Languages

  • TLA+ (Temporal Logic of Actions): Developed by Leslie Lamport, TLA+ is used to model concurrent and distributed systems. It is an industry standard at companies like Amazon (AWS) for verifying complex protocols.
  • Alloy: A declarative specification language for expressing structural properties and behavior. It uses SAT solvers to automatically find counterexamples to a design.
  • B-Method / Event-B: Used extensively in the railway industry (e.g., Alstom, Siemens) to prove the safety of automated train control systems.

AI-Assisted Verification

We are researching how agents can automatically generate TLA+ or Alloy specifications from high-level goals. This allows us to use Model Checking to prove that an agentic workflow is deadlock-free and safe before deploying it.

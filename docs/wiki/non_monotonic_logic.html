The Logic of Revisable Beliefs

Non-monotonic logic challenges a core assumption of classical logic: that once a conclusion is derived from a set of premises, adding more premises can never invalidate it. Human reasoning clearly violates this principle. We routinely withdraw conclusions when we learn new information (“birds fly” until we learn that this particular bird is a penguin).

Non-monotonic formalisms were developed to capture this behaviour. Instead of treating knowledge as a growing, never-corrected set of theorems, they model default assumptions, exceptions, and revision policies. This makes them especially suitable for domains where rules are true “in general” but not “always”, such as law, medicine, or everyday common-sense reasoning.

Non-Monotonic Reasoning in AI

In artificial intelligence, non-monotonic logic provides the tools to represent defaults (“normally, X holds”), defeaters (“except when Y is true”), and priority between conflicting rules. Frameworks such as default logic, circumscription, and answer-set programming all embody variants of this idea.

From a systems perspective, the key requirement is the ability to revise conclusions deterministically when theories change, while keeping a clear audit trail of what changed and why. Non-monotonic reasoning should not feel like ad-hoc patching; it should be principled and inspectable.

Non-Monotonic Behaviour in AGISystem2

AGISystem2 implements non-monotonic behaviour through theory layering and validation. Instead of mutating base concepts, the engine maintains a stack of theory layers: base facts, domain overrides, contextual policies, and temporary counterfactual layers. A conclusion is always evaluated with respect to the currently active stack.

When new information arrives—such as an exception, a stricter policy, or a temporary scenario—the system does not “forget” old rules. It adds a layer on top that can narrow or reshape the relevant concept volumes. The TheoryStack component controls the order of layers, their activation, and how conflicts are detected.

To keep this behaviour safe, the ValidationEngine performs abstract checks: it can detect when an added layer makes previously valid combinations impossible, when two layers clash irreconcilably, or when a seemingly benign override would break guarantees. The BiasController can also apply masks to prevent certain dimensions (for example protected attributes) from influencing revised conclusions.

Philosophical and Practical Implications

Non-monotonic reasoning in AGISystem2 reflects a more realistic picture of knowledge: not as a frozen catalogue of facts, but as a layered structure subject to revision and contextual re-interpretation. This is essential in domains where regulations change, exceptions proliferate, or competing value systems must be reconciled.

Practically, it also protects users from overconfident but brittle answers. By design, the system can say “previously I would have answered YES, but under this new policy layer the answer becomes NO, and here is why”. That shift—from opaque rule firing to transparent, layered revision—is central to building trust.
